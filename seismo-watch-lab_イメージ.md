# 世界の地震データ取得・日本語化Webアプリ構成案

海外の地震データを日本語環境でスムーズに検索・取得できる静的Webアプリ（SPA: Single Page Application）の構成案です。サーバーを必要とせず、ブラウザ上で完結するため維持費がかからず軽量に動作します。

## 1. システムの全体構成（技術スタック）

* **フロントエンド**: JavaScript (Vanilla JS または React/Vue.js など)
* **スタイリング**: Tailwind CSS などのユーティリティクラス（UI構築の高速化）
* **ホスティング**: GitHub Pages, Cloudflare Pages, Vercel など（無料かつ静的ファイルの公開に最適）
* **データ通信**: Fetch API を用いた非同期通信

## 2. データ取得元（API）

世界標準であり、無料でアクセスできるAPIを利用します。

* **USGS（アメリカ地質調査所）Earthquake API**: 
  * 世界中の地震データをGeoJSON形式でリアルタイムに提供しています。
  * マグニチュード、発生時刻、震央の緯度・経度、深さなどの検索パラメータをURLに付与して必要なデータを絞り込めます。

## 3. 日本語化（翻訳）のアプローチ

動的な翻訳API（DeepLやGoogle Translate）を使用するとAPIキーの保護（バックエンド処理）が必要になるため、静的アプリでは**辞書マッピング**によるアプローチが最適です。

* **UIと専門用語の固定翻訳**: アプリ内のメニューや、APIから返ってくる固定のパラメータ（Magnitude → マグニチュード, Depth → 深さ, Location → 震央など）は、JavaScript内の辞書オブジェクトを用いて自動的に日本語へ置換します。
* **地名のローカライズ**: USGSのデータには「10km WNW of Tokyo, Japan」といった地名が含まれます。これを完全に翻訳するのは難しいため、「Japan」→「日本」、「Taiwan」→「台湾」といった主要国・地域のキーワード検知による部分翻訳を実装すると、実用的な視認性が得られます。

## 4. データのダウンロード機能

ブラウザの標準機能を活用し、取得したJSONデータを解析や構造計算などに使いやすい形式に変換して保存させます。

* **Blob APIの活用**: JavaScript内でデータをCSV文字列などに変換し、`Blob` オブジェクトを生成します。これを一時的なダウンロードリンク（`URL.createObjectURL`）に紐付けることで、サーバーを介さずにローカル環境へ直接ファイルをダウンロードさせることができます。
* **エクスポート形式**: Excel等で扱いやすい **CSV形式** と、プログラムで再利用しやすい **JSON形式** の両方をサポートします。

## 5. 想定される画面UI（操作フロー）

1. **検索パネル**: 期間（〇年〇月〇日〜）、最小マグニチュード、対象地域（緯度・経度の範囲）を日本語UIで入力。
2. **データ取得**: 「検索」ボタン押下で USGS API にリクエストを送信。
3. **結果一覧（日本語表示）**: 取得したGeoJSONデータを解析し、テーブル形式で一覧表示。列名はすべて日本語に変換。
4. **ダウンロード**: 一覧の下にある「CSVでダウンロード」ボタン等を押すと、表示されているデータがそのままファイルとして出力される。
